## 用語定義

以下の用語をユーザーへの説明に使うこと。文脈に応じて使い分け、ゆらぎを避けること。

| 用語 | 使う文脈 | 意味 | 初出時の補足 |
|---|---|---|---|
| リポジトリ | GitHub・Claudeコード の操作 | クラウド上のファイル保存場所 | 「リポジトリ（ファイル保存場所）」 |
| プロジェクト | Claudeチャット（claude.ai）の操作 | Claudeチャット上の作業単位。リポジトリと1対1で対応する | 「プロジェクト（Claudeチャット上の作業単位）」 |
| 作業フォルダ | 共通 | リポジトリ内の個別作業単位 | 「作業フォルダ（提案書作成、報告書作成など）」 |
| Claudeコード | 共通 | ファイルの作成・編集を行うブラウザ画面 | URL: https://claude.ai/code |
| 接続キー | 共通 | Claudeチャットがリポジトリにアクセスするための認証情報 | 正式名称: Personal Access Token |

**使い分けの原則**: Claudeチャット（claude.ai）の操作を説明するときは「プロジェクト」、Claudeコード（claude.ai/code）およびGitHub（github.com）の操作を説明するときは「リポジトリ」を使う。Claudeコードはリポジトリに直接接続するため、「プロジェクト」は使わない。

## 環境要件

セットアップに必要なもの。不足がある場合はその場で取得・登録を案内する。

1. **メールアドレス**: GitHubアカウントの作成に使用。普段使いのもので可
2. **claude.ai の有料アカウント（Pro プラン以上）**: Claudeコード とプロジェクト機能に必要。未加入の場合、画面左下のメールアドレス→「プランをアップグレード」から登録（月額約3,000円〜）
3. **Webブラウザ**: Google Chrome を推奨

## セットアップTODO

上から順に実施する。順番の変更は不可。
各TODOに「完了条件」を記載している。完了条件を満たしたことを確認してから次に進む。

### TODO 1: GitHubアカウントの作成

- **目的**: ファイル保存先となるGitHubのアカウントを作る
- **URL**: https://github.com → 「Sign up」
- **すでに持っている場合**: ユーザー名を聞いてスキップ
- **完了条件**: GitHubのユーザー名（メールアドレスではなく英数字の短い名前）を取得・記録

### TODO 2: リポジトリの作成

- **目的**: テンプレートからユーザー専用のPrivateリポジトリを作成する
- **URL**: https://github.com/tatugmad/claude-office-template → 「Use this template」→「Create a new repository」
- **設定**:
  - Repository name: `OfficeWork` を推奨（半角英数字のみ、スペースなし）
  - 公開範囲: 必ず **Private**
  - Include all branches: チェックしない
- **背景説明（ユーザーへ）**: リポジトリはクラウド上のファイル保存場所。この中に作業フォルダを作っていく。日常業務は1つのリポジトリにまとめて作業フォルダで分け、大規模な別業務は別のリポジトリを作ることを推奨
- **完了条件**: リポジトリ名を取得・記録

### TODO 3: 接続キーの発行

- **目的**: Claudeチャットからリポジトリを読み書きするための認証情報を発行する
- **URL**: https://github.com/settings/tokens?type=beta → 「Generate new token」
- **設定**:
  - Token name: 任意（例: `claude-access`）
  - Expiration: Custom → 365 days（1年）推奨
  - Resource owner: 自分のアカウント
  - Repository access: 「Only select repositories」→ TODO 2で作成したリポジトリを選択
  - Permissions → Repository permissions → Contents: 「Read and write」。他は初期値のまま
- **重要注意**: 画面を閉じると二度と表示されない。必ずコピーして安全な場所に保存
- **完了条件**: 接続キー（`github_pat_` で始まる文字列）をチャットに貼り付けてもらい、記録

### TODO 4: Claudeチャットのプロジェクト作成と接続情報の登録

- **目的**: claude.aiにプロジェクトを作り、接続キーとリポジトリ情報を登録する。登録後は、このプロジェクト内のClaudeチャットでClaudeが自動的にリポジトリへアクセスできるようになる
- **URL**: https://claude.ai → 左サイドバー「プロジェクト」→「新しいプロジェクトを作成」
- **登録内容**: 「プロジェクトの指示を編集」に以下を貼り付ける。TODO 1〜3で記録した実際の値を埋め込んだ完成形テキストをClaudeが生成してユーザーに提示すること（プレースホルダ不可）:

```
## 環境識別
CLAUDE_ENV=chat

## リポジトリアクセス

GitHub Personal Access Token（Fine-grained、Contents Read-write）:
{接続キー}

対象リポジトリ:
- {ユーザー名}/{リポジトリ名}（Private、本番）

セッション開始時、リポジトリのCLAUDE.mdを読み込むこと。
トークンの有効期限切れと思われる場合、対応を指示すること。
```

- **完了条件**: プロジェクトが作成され、指示が保存されたことをユーザーが報告

### TODO 5: Claudeチャットでプロジェクトに入る

- **目的**: TODO 4で作成したプロジェクトを選択し、Claudeチャットからリポジトリにアクセスできることを確認する
- **手順**:
  1. **このチャットは開いたまま**、ブラウザの別タブで https://claude.ai にアクセスする
  2. 左サイドバーの「プロジェクト」から、TODO 4で作成したプロジェクトを選択する
  3. プロジェクト内で**新しいチャット**を開く
  4. 「リポジトリにアクセスできるか確認して」と入力する
  5. Claudeがプロジェクトの内容について応答すれば成功
- **確認方法**: 画面上部またはサイドバーにプロジェクト名が表示されていれば、プロジェクト内にいる。プロジェクト名が表示されていない場合は、通常のチャットになっておりリポジトリにアクセスできない
- **完了条件**: Claudeがプロジェクトの内容について応答したことをユーザーが報告

### TODO 6: Claudeコード でリポジトリに接続する

- **目的**: Claudeコード でリポジトリを開き、正常に動作することを確認する
- **手順**:
  1. **このチャットは開いたまま**、ブラウザの別タブで https://claude.ai/code にアクセスする
  2. 初回はGitHub連携の許可を求められる → 「Authorize」をクリック
  3. 画面下部の入力欄の左下に「+」ボタンがある。クリックして、TODO 2で作成したリポジトリを選択する。選択後、ボタンにリポジトリ名（例: OfficeWork）とブランチ名（main）が表示される
  4. 入力欄に「このリポジトリの内容を確認して」と入力して送信する
  5. Claudeがリポジトリ内のファイル一覧や内容について応答すれば成功
- **確認方法**: 入力欄の左下にリポジトリ名（例: OfficeWork）が表示されていれば、正しく接続されている。表示がない場合は「+」ボタンからリポジトリを選択すること
- **完了条件**: Claudeがリポジトリの内容について応答したことをユーザーが報告

### TODO 7: 確認と注意事項

- **目的**: 今後の利用で必ず知っておくべき注意事項を説明する
- **完了条件**: ユーザーが理解したことを確認

以下の内容を説明すること:

**うまくいかない場合:**
- Claudeチャット・Claudeコード のどちらかでうまくいかない場合は、**このチャットに戻ってきて状況を教えてください**。一緒に解決します。

**重要な注意:**
- 新しい会話を始めるたびに、**Claudeチャットではプロジェクトを、Claudeコードではリポジトリを毎回選択する**必要がある。選択を忘れるとClaudeはリポジトリを読めず、通常のチャットと同じ動作になる
- ClaudeチャットもClaudeコードも、新しいチャットやセッションを開いたら、**何か一言入力して送信するまでClaudeは動かない**。入力すると、Claudeが自動的にリポジトリの設定ファイルを読み込み、作業を始められる状態になる
- 使い方について詳しく知りたいことがあれば、Claudeチャットのプロジェクト内でClaudeに質問すれば回答できる。Claudeはリポジトリ内の設定ファイルを読んでいるため、あなたの環境を理解した上で対応する
